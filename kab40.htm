Pipeline Delivery - Pipeline Delivery Tools for Kubernetes

 

Now that you have developed successfully under an Appsody development environment, including the use of Codewind to make things easier for testing, it is time to put that finalized Appsody Build into the IBM Cloud inside a Kubernetes Cluster.  If you recall, in the previous section, you created a Docker Image from your Appsody development environment, which can be seen below in the local Docker Repository.

 

docker images

 

 

You must get that image over into your IBM Cloud Registry so it can be used by an IBM Toolchain to place inside your Kubernetes Cluster that was created in the previous course.

 

Step 16

 

You must login to your IBM Registry in preparation for pushing your dev.local/sample2cloud image.  You will be pushing an alias of it from your local Docker Repository to your IBM Cloud Registry. 

 

First, login into the IBMCloud

 

ibmcloud login -a https://cloud.ibm.com -u ibmid@email.com

 

You must ensure your region is set correctly.

 

ibmcloud cr region-set us-south (or whatever region you are in)

 

Remember, you created a namespace prior that looked something like ibmID-myimages.   Put your image into the correct format for the IBM Registry by creating an alias.  Replace the word namespace with your created namespace (Quick2Cloud uses pfk-images you use yours).

 

docker tag dev.local/sample2cloud us.icr.io/namespace/sample2cloud

 

List out your docker images and you will see us.icr.io image alias

 

docker images

 

 

Step 17

 

Push your alias into the IBM Cloud Repository.

 

docker push us.icr.io/pfk-images/sample2cloud

 

Step 17A (skip to 17b if the push succeeded)

 

Note:  Your IBM Cloud Registry, with the plan you are on, has limited space available for about five registry images.  If the push fails do to space limitations, you will have to remove your namespace and create it again.

 

ibmcloud cr namespace-rm namespace

ibmcloud cr namespace-add namespace

docker push us.icr.io/namespace/sample2cloud

 

Step 17B

 

List out the new image in the IBM Cloud Registry

 

ibmcloud cr images

 

 

Pipeline Delivery

 

As you remember from your previous course, an IBM Toolchain was used to place your sample application into the IBM Cloud, either as a process or a Container in a Kubernetes Cluster.  If you think back to course 100, you used a simple push to get your application from your desktop directory into the IBM Cloud.  That push was really a simple Pipeline Delivery mechanism IBM Cloud provided - but that was not clear to you at that time.  From what you know now about getting an application in the Cloud, it requires some type of Pipeline Delivery.

 

To get your Appsody Kabanero work into the IBM Cloud, you can just create (or reuse) a Toolchain like you did in the previous course.  Quick2Cloud has repeated that step from the previous course below.

 

Step 17C

 

You will configure the Deploy Stage in your Kubernetes Toolchain (either create a new on or reuse your last Toolchain from the last course) to deploy the image created by using Appsody with the Kabanero image.

 

Lets configure the deploy tool from the Toolchain you created in your previous course for getting your Docker image into Kubernetes.

 

 

In the configuration display below, make sure the three sections are still configured correctly.

 

 

If the Toolchain is not showing the correct configuration, then use the information below to reconfigure it.   Otherwise, just verify all the names in the script are correct.

 

Image name could look like sample2cloud

Image tag could look like latest

Registry URL could look like us.icr.io (or whatever IBM assigned you)

Registry namespace could look like ibmID-myimages (or whatever you picked)

Pipeline Cluster name could look like ibmID-mycluster (or whatever you picked)

 

 

New Deploy script below. Cut and Paste everything in green and substitute everything in purple.

 

IMAGE_NAME=your-application-name

IMAGE_TAG=your-application-tag

REGISTRY_URL=us.icr.io

REGISTRY_NAMESPACE=your-namespace-name

USE_ISTIO_GATEWAY=

KUBERNETES_SERVICE_ACCOUNT_NAME=

KUBERNETES_MASTER_ADDRESS=

KUBERNETES_MASTER_PORT=

KUBERNETES_SERVICE_ACCOUNT_TOKEN=

IMAGE_NAME=your-application-name

IMAGE_TAG=your-application-tag

REGISTRY_URL=us.icr.io

REGISTRY_NAMESPACE=your-namespace-name

PIPELINE_KUBERNETES_CLUSTER_NAME=your-cluster-name

CLUSTER_NAMESPACE=prod

source <(curl -sSL "https://raw.githubusercontent.com/open-toolchain/commons/master/scripts/check_and_deploy_kubectl.sh")

 

Click on Save at the bottom

 

Lets run Deploy again. This will place your Docker Image as specified in the script above, into the Kubernetes Cluster bypassing the Build and Containerize Stages. Click on the button below in yellow to re-run the Deploy stage.

 

 

 

You can use Deploy in this way to put any Kabanero Image you create with Appsody commands into a Kubernetes Cluster. By editing the script above, substituting another Docker image name from the IBM Cloud Registry, that image too can be set to run as an additional Container.

 

Tekton  a Kubernetes native Pipeline Delivery Tool

 

There is another pipeline delivery tool getting popular for placing Containers into Kubernetes Clusters.  Tekton runs as a Container in the Cluster and works to get your application into the Cluster as well.  What is different with Tekton is that it is a Kubernetes based offering which means it can be leveraged in any Cloud environment instead of using of relying on a specific Cloud developed Pipeline tool.

 

IBM Clouds use of Tekton, at the time this course was developed, was in Beta and will be updated in Quick2Cloud as soon as it becomes available.  Until then, you can use the above Toolchain to deploy your Appsody built Kabanero image into your Kubernetes Cluster.
