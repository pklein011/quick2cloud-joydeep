Automatically Scale Instances of your Application

 

Fantastic job so far, you have gotten the CF-Community defined and have access to all the features that community has to offer including the one we are most interested in for this course - Auto-Scaler.   As you probably figured out, Auto-Scaler is a cf plugin that contains its own set of CLI commands such asa, aasp, asm and ash; all of the commands that we will eventually get to use.

 

IBM Cloud currently provides about five different performance metrics that it measures and makes these metrics available for Auto-Scaling as triggers so you can be critical to which bit of application performance you want to Expand or Contract your instances with.  They are:

 

1.       Memory Used (in mb)

2.       Memory Utilization (% of total mb)

3.       CPU (% /sec)

4.       Response Time (requests/ms)

5.       Throughput (requests/sec)

 

Step 6

 

Let us begin my defining an Auto-Scaling Rule that governs when you automatically expand an instance of your application.  This rule applies to both an application in the IBM Cloud as well as a Continuous Delivery Toolchain that is running your application.

 

For example, the default rule shown below for Expansion is:

If (Percent of memory used) > 80 then increase an instance but never increase past 2 instances.

 

For example, the default rule shown below for Contraction is:

If (Percent of memory used) < 30 then decrease an instance but never decrease less than 1 instance.

In setting the above rule, the threshold must be violated for at least 120 sec before it is considered triggered.  Also, after being triggered, it will not be tested again for another 120 sec.  Once the trigger has officially happened one instance will be added or removed until the next test takes place.

 

Note: You can modify the testing time periods and number of instances to involve.  This will be shown later in the course.  For our work right now, the default trigger times plus single instance increments will suffice.

 

 

Step 7

 

Quick2Cloud will now generate a script that will define to the IBM Cloud the rules that you just created.  This script must be run in the BASH Command Line Console (it will not work in the DOS or PowerShell consoles).  This is one of the few times that BASH is absolutely required.

 

Generate: college-auto-scaler-script

 

Change your directory to q2c-web-app and execute the script as shown below.

 

 

This will create a file in the q2c-web-app directory in the naming format:

 

qvsample-auto-scaler-policy.txt

 

The content of that file will be pushed to the IBM Cloud and enabled for your qvsample application.  After this step Auto-Scaling will be in use on your Quick2Cloud Sample Web Server Application.

---

Removing an Auto-Scaling Rule

 

Should you want to remove your Auto-scaling Rule from your application for the purpose of updating or just deleting, then the following command detaches it from your application paving the way for a new one.

 

cf  detach-autoscaling-policy qvsample

 

Upon completion, you can restart at Step 6 and create a new Auto-Scaling Rule.
