Automatically Scale Instances of your Application

 

Having a script containing your Auto-Scaling Rules added to your application’s local directory is very important making your application portable to other region deployments just by resetting your endpoint and re-issuing all your scripts again.  Currently, you only need to manually create two Toolchains in whichever endpoint you want to run in and use the scripts to do the rest.

 

There is another way to define Auto-Scaling Rules and that is by using a new Auto Scaling facility in the IBM Cloud Dashboard.  Using that Dashboard feature allows for a bit more control over your rules if you don’t mind having to manually create them for each deployment.

 

Using the IBM Dashboard for Auto Scaling

 

Step 0

 

Log into the IBM Dashboard at www.ibm.com/cloud and select your application from the Cloud Foundry resource display.

 

 

Step 1

 

Select AutoScaling from the options on the left side of the display

 

 

Step 2

 

Click on Create Auto-Scaling Policy.  Fill in the Auto-Scaling form.

 

 

Step 3

 

Using the IBM Cloud Dashboard you can also define schedules for when your application should have lots of instances and schedules when it does not need a lot of instances.  This course focuses on the performance related Auto-Scaling features but if you also desire scheduling use the IBM Cloud Dashboard for this purpose.

 

 

Super job!  You understand Auto-Scaling and have it ready to work for you.
