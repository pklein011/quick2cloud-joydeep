Craft an Application for the IBM Cloud

 

Welcome to college, let us try and pick up the pace a little.  For your purpose here in college you will be deploying a Quick2Cloud Sample Web Server Application into the Cloud to get an idea how the whole deploy thing is accomplished.  This will be a local deploy into the Cloud, meaning, it will use as its code source a desktop directory where Quick2Cloud will generate an application for your selected runtime. 

 

Tip: Before you start.  Create a Web Browser Shortcut to quick2cloud.org so you can enter Quick2Cloud quickly from the Desktop.

 

 ==>

 

Later, you will have the choice to deploy this sample application from not only a directory but also from a locally residing code repository, a remote repository located at Github.com, or a repository located in the IBM Cloud itself.  You may choose to just maintain your code locally in a desktop directory, specifically if you are the sole developer whose actions will not get in the way of others.  If you choose to house your code in a directory, you will be coding and potentially testing it on the desktop then pushing it up to the IBM Cloud as a place to run for the entire world to see.

 

Tip: Quick2Cloud College uses color to help highlight key concepts of what is being discussed.  Bold or purple are items that need to stand out where green is for everything repository related such as the name of a repository command.

 

The required use of a repository comes into play when you want to develop and test in the Cloud or are working with a team of developers where a repositories source code management strategy stops everyone from getting in the way of each other.   Also, the use of a repository will become essential later in your courses when you want to understand some of the more contemporary Cloud technologies guiding the industry today.  Finally, using a repository makes you more marketable since most large companies have adopted the use of repositories for some time.  Look at any job posting of today and you will see the requirement of understanding repositories listed at the very top.    It is a good idea to at least practice using a repository, just for your professional growth if for nothing else, and that will be learned when Course 103 is taken. 

 

But, for now, as you begin understanding how the Cloud works, we march forward in this course with the source code residing locally on a desktop in an application directory.     

 

Let us begin

 

With Quick2Clouds assistance, you will create a local directory that will hold Quick2Clouds generated sample application and once Quick2Cloud comingles it with the necessary Cloud support files, it will ready to be moved (pushed) up into the IBM Cloud. 

 

Diagram of what you are about to accomplish:

 

 

 

In this course, your local source code directory will be called q2c-web-app and it will be the target of all Quick2Cloud assist operations.   Keep this directory name just as stated but you can put it anywhere you want.  You may want to put it somewhere easy to get to, such as in the Root since it will be referenced a lot during your college days.  Primarily, q2c-web-app will be the target of downloaded applications, scripts and files that Quick2Cloud will be generating for you and ultimately it will contain a working sample web server application.

 

While at college, you will be following Steps, performing each one in order, with an explanation of what that step is accomplishing.  When questions are asked by the Quick2Cloud user interface that requires you to enter in something into an input field, click the gray button that is in close proximity so that information can be transferred to the Quick2Cloud Server. 

 

It is very important not to skip answering any of the questions or clicking of the associated buttons before moving on or your application deployment may fail.  Your choices will be remembered and show up in the fields as long as you dont close your browser.  If you do close the browser, just reenter the questions again. 

 

We always start at Step 0 in our college courses, why not, you are all developers and indexing from zero should feel normal by now.

 

In assisting you, it is the intention for Quick2Cloud to eventually support various Cloud programming runtimes and languages and depending on which one you pick will depend on how this course provides information.  For example, you are defaulted to Node.js in the initial courses but after graduation you will be free to pick others such as Python and you may find you will be asked a different set of questions to make that application deploy.  Then again, you may select containerization with Docker and that will cause yet another set of questions and assists.  To not overwhelm you at this time we will default to an SDK for Node.js runtime for Course 100.

 

Enter your IBM ibmID that you created when signing up for IBM Cloud.  The format is ibmID@email.com.  Substitute your email address in place of what is showing.  Then click on Set UserID
