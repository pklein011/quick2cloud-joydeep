Tail your Applications Log in a DOS Prompt

 

Once your application is running in the IBM Cloud it would be to your advantage to see its log messages without having to log into the IBM Cloud Dashboard.  This is especially useful if you are operating your modified Quick2Cloud sample web application waiting for any problems to occur.  Having a DOS Prompt that is dedicated to only application messages is a nice way of compartmentalizing those messages off to the side of your display. 

 

You can also have several DOS Prompts displaying messages from several applications for comparing messages when debugging systems that communicate with each other.  This will become very important when your college talks about creating micro-services in the 200 Series Courses.

 

You must be logged into the Cloud for this to work.  If in issuing any of the commands below you get a message that you are not logged in to the IBM Cloud then issue the following:

 

Step 0

 

cf login -u ibmID@email.com

 

Replace your ibmID@email.com with your real ibmID you got when you signed up for an account.

 

Then, it will prompt you for a password

 

Step 1

 

Get a list of the applications you have running so you can pick the exact ones you wish to see log messages for.   This is accomplished by issuing the following command:

 

cf apps 

 

 

Notice, in the above list of applications, some of the applications will be from a simple deploy such as qvsample or it could be more advanced from a Toolchain as is the case above.  The application in the above shell is called simple-toolchain-quick2cloud-production

 

Step 2

 

You can either tail the application log (see all new messages as they occur, when they occur) or ask to see the most recent messages (a handful that happened sometime in the past).  To see the most recent messages your application issued, type the following but using your application name:

 

cf logs simple-toolchain-quick2cloud-production --recent

 

 

From the messages above, you are seeing one of the most recent HTTP headers that had been logged out.

 

Step 3

 

To tail the message log and see all messages that gets created from this moment forward, issue the following:

 

cf logs simple-toolchain-quick2cloud-production 

 

 

Tuck this DOS Prompt off to the side of your desktop and as your application in the Cloud is used, you will see the messages that get created.  Until then, it will just sit there waiting for a message to arrive.

 

You can issue this cf logs command for as many applications as you want!
