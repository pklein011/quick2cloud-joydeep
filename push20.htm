Add Push Notifications to Android Mobile Devices

 

Android Studio

 

As the second part of enabling IBM Push Notifications, you must download and install Google’s Android Studio and SDK.  This is where you will develop, build, run and test your Android Application in an Android Device Emulator before running it on a physical Android Device.  The studio is a very large and comprehensive facility and Quick2Cloud will show you just the parts needed to get push notifications working.  By the time you are done syncing, configuring and running your Android Application you will have a good grip on how this facility works which will allow you to create other applications as well.  So, after saying all that, let’s get started.

 

Step 0

 

You will find the download of the Android Studio at https://developer.android.com/studio for your Windows system.  After downloading, install the application and place an icon on the Desktop or in the system tray. 

 

Step 1

 

Start Android Studio and stop when you get to the display below:

 

 

Arriving at this display indicates that Android Studio was installed correctly.  For now, we will leave this and move on to the next step in our push notification process but will return to the Studio shortly.  Keep it out of the way for now.

 

Step 2

 

IBM Push Notification Service

 

Log in to IBM Cloud at www.ibm.com/cloud

 

IBM provides a service that allows a Cloud Service to send notifications to a mobile device.  To acquire this service you will go to the IBM Service Catalog and select Push Notifications from the list as shown below.

 

 

Click on Push Notifications and choose the Lite Plan.

 

 

Create the Service my clicking on Create and accept all the default names and options.

 

Step 3

 

On the IBM Dashboard you will see your Service in the Apps window.  This will be the entry point into your Push Notification Service each time you need to reference it.

 

 

Click on Basic CVQUS to begin using the Service.

 

 

Upon entry, you will be presented with the above display.  Notice IBM assigned you three keys that are highlighted in yellow.  These will be used in configuring your Android Application in the Android Studio.  For now, cut and paste the keys to a file or you can return here when configuration time comes.  Under the getting started quickly, you don’t have to read the README.md file for Quick2Cloud has taken care of those steps for you.

 

Now, to finish installing the IBM Push Notification Service, download the sample Android Application code by clicking on the Download code button above.  Unzip it to a directory called q2c-mobile.  The directory should look something like this:

 

 

Congratulations!  You have created your IBM Push Notification Service and it is ready to use.  But first, some additional configuration must be done to enable it for operation.
