Step 1

 

Generate and download the Quick2Cloud Sample Web Server Application and extract it into the q2c-web-app directory created in the previous step.  Quick2Cloud will generate you a web server with basic function to host web pages, deliver files, manipulate Cookies, connect to a database Service and provide access to Cloud Environment Variables when you are running in the Cloud or locally on the desktop. 

 

Although this is a sample application, it is designed for you to make it the base of your own project.  Modify it or create another from scratch, basically do whatever you like to create your own thing.   Quick2Cloud will provide time for you to personalize it later in the college curriculum.  However, if you have not boned up on your coding skills for some time; that is ok as well.   You can either use the sample application ASIS or add some personalization code by reading the node.js getting started documentation as a guide.  The flexibility is yours to decide.

 

The sample application is filled with comments, extendable functions and as a first project very useful in exploring the IBM Cloud features.  It is designed to help in the understanding Cloud nuances, such as how to use the CFENV service for manipulating environment variables and the Cloudant and Watsons Services for how to manipulate an IBM database and language translations.  Study the sample code in hope that it will explain some of the more cryptic workings you will be dealing with.   More on those topics will follow in upcoming courses.

Step 2

 

To operate in IBM’s Cloud your application must carry several Cloud support files that instruct the Cloud on how to proceed with deployment.  These files are known as the Manifest, Package and Procfile.

 

Manifest.yml

 

Package.json

Procfile

 

These are supplied as part of the download, complete with default values, but you can change them, after graduation, by editing these files to alter some of the runtime parameters.  For example, if you add much code to the sample application and it grows past its initial 128M memory size, you can edit the Manifest.yml and bump it up to 256M. 

 

Note: Leave manifest.yml at 128M for our college course.

 

In addition, these other Cloud support files found in the node_modules directory contain Node.js functions that are required for the application to run.  They are part of the information to be given to the Cloud with your application.  In our case, Quick2Cloud will generate these two Node.js functions as part of your next download to use when you run the application on the desktop.  The Cloud does not push them to the Cloud when it comes time to run in the Cloud. Instead, the Cloud will generate them automatically for you as part of the Cloud’s Build and Deploy process.  You download them primarily for running your application locally on the desktop.

 

 

 

F.Y.I.

CFENV is the Cloud Environment Variable processing function that retrieves the Cloud information about the application and Services that have been bound to your application (Services are covered in the next course).  For example, your application will find the URL to access the sample application, userID and password for a bound database and any other items that are required for your application to run properly.  Later, you will see that this same CFENV function will work locally when you develop and test on your desktop (Course 102).

 

VCAP_SERVICES is an alternate function that also allows retrieving the Cloud environment variables.  Your Quick2Cloud sample application can use either one; it will become your choice.  For this course the decision was made to use the CFENV facility so you will see code snippets of its implementation in the sample application code.  After graduation, you can choose VCAP_SERVICES if you so desire.  Many other Node.js functions you choose to use after graduation will reside in this special node_modules of q2w-web-app as well.

 

If you use the NPM Install to obtain these facilities, they will appear in that node_modules directory after the install runs.

 

Note: NPM is a facility that houses many popular Node.js functions where you can install from.  They are called packages.
