Add Push Notifications to Android Mobile Devices

 

Android Studio and IBM Push Notification Configuration

 

You will now tackle the task of pointing your IBM supplied Sample Android Application to the IBM Push Notification Services.  Also, you will get the Gradle Plug-in configured as well as the Android Studio SDK to be used in conjunction with Android Studio.

 

Step 0

 

Locate the minimized Android Studio that you installed prior or start it by clicking on its icon.  When it comes up you should see the following:

 

 

At this point you will Open an existing Android Studio project which is the IBM Sample Android Application located at q2c-mobile.  Making this selection will begin a process of getting Android Studio to take your sample application under its control for syncing, building and running.

 

Step 1

 

Once Open is selected, you will need to select the q2c-mobile directory and proper sub-directory for Android Studio’s use.

 

 

Click on OK and Android Studio begins a set of initializations that occurs on its very first invocation.  The moment your IBM Sample Android Application is embraced by studio it will begin the process of syncing your Gradle Files.  However, because this is the first invocating, Studio will automatically install the Gradle Plug-in which will take a little extra time.  Look for the Gradle messages in the bottom part of the Android Studio window as shown below:

 

Android Studio

 

Again, when you select your Android Project, Gradle will automatically sync your Gradle Build files and you will see the messages in the bottom window.  If you get error messages in this window you will need to fix them before moving forward.  As in the case above, you may see warning messages, in this case, Android Studio automatically fixed these problems for you as the messages imply.   It is demonstrated to you this way because at some point you will inevitably generate error messages and need to understand how to fix them.  The messages above indicate that the commands compile and testCompile have been deprecated and have been automatically replaced with the api and testimplementation commands.  It is best that you go into the Gradle Build files and replace those old commands with their new ones even though Android Studio fixed them on the fly for you.

 

Because IBM push Notification provides Gradle Build files and Google’s Firebase requires that their commands be added as well, getting Gradle to sync files can be a bit of an art.  Quick2Cloud has attempted to provide you up to date Gradle Build files, that is what you downloaded prior, so you should have little issues with the Gradle Sync task.  Make sure you have a clean sync, free of severe error messages, before moving to the next step.

 

Should you need to edit a Gradle Build file, in attempt to fix a sync error, use the following to invoke the Gradle Sync again:

 

 

Keep fixing the Gradle Files by selecting them from the left side tree, edit, then re-invoke the sync until you achieve success.  Hopefully, that will occur on the first try.

 

Step 2

 

Now that Gradle has successfully synced, you will need Android Studio to download the Android Studio SDK.  The easiest way to accomplish this is to do the following:

 

Click on Help then Find Action…

 

 

Type in SDK Manager then click on the SDK Manager topic that appears.

 

 

Then you will get the SDK Manager display.  Select the Android SDK from the left side menu and then click on the latest Android version.  Click on OK then Android Studio will download and install the SDK.

 

 

Note: Once downloaded, the SDK will reside, in Windows, under Users\userid\AppData\local\Android but keep in mind that AppData is a Windows hidden directory so you will have to unhide it to see the SDK.  However, Android Studio will find it hidden or not.

 

Step 3

 

Wow, so much configuration and you still are only three quarters of the way done.  Your next step is to create a mobile device emulator so you can test out push notification on your laptop before applying it to your physical mobile device.  You want to do this because it makes testing very easy, you can try your Android App on any kind of device to see how it works and you are guaranteed it is bug free before exposing it to your mission critical mobile device.  Android Studio allows you to create one or more emulators by doing the following:

 

Click on the pull down in yellow below, then click on Open AVD Manager.  Notice, in this example, there are two emulators already defined (Nexus and Pixel) and you will be defining another.  However, the first time in you won’t see any preconfigured emulators.

 

 

You will then see your virtual devices that have been already defined.  Again, if this is your first time in, you won’t see any.  Click on Create Virtual Device to create one.

 

 

Pick the device characteristic closest to your mobile device then make it your selection in the pull-down window as shown below:

 

 

This says your mobile emulator is a Nexus 5X.

 

Step 4

 

OK, almost there.  Next you will configure your IBM Supplied Android Application to point to your IBM Push Notification Service.  This is done by editing the IBM Supplied Android Application’s Java program and adding the IBM Push Notification credentials and userID.

 

 

Take careful look at the left side tree above and see the deep path to the MainActivity java application.  When selected that application source code is shown in the window on its right.  Bring up your IBM Push Notification credentials and edit in your IBM Push Notification appGuid and clientSerect credential.  Also, add in your IBM userid in the appropriate location.   Once again, if you did not save the credentials you can go to your IBM Push Notification Service and find them there.  By doing this, your supplied IBM Android App, upon starting, will register your device and userID with IBM Push Notification and will do it successfully because you carefully gave it the right credentials.  Any mistake or typo here will make the registration fail.

 

Fantastic!  You have Android Studio and your Android App configured.  You have a few more configurations to finish before you can test out sending your mobile device a message.
