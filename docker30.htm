Porting Sample Application to Docker Containers

 

Verifying the Application Container

 

Step 6

 

Taking another look at your running Docker Containers, you will see a listing of the Port mappings.

 

docker ps

 

 

Notice that 49160 is the Container Port used to communicate with your sample application at Port 8080.  This number was assigned on the Docker Run command and is designed to be unique across all Containers allowing you to have more than one Container application using the same 8080 Port.  Think about the power in that Docker feature, especially if you run Containers from third party images and you have many images using the same Port.  Luckily, you do the Port assignment yourself when you start the Container which avoids any conflict.

 

Step 7

 

To verify your Quick2Cloud Sample Application is working do the following in a web browser:

 

localhost:49160

 

 

Excellent!  You now have your sample application proven to be running on your Desktop in a Docker Virtual Container.

 

Step 8

 

It is possible in Docker for you to instantiate a command line interface into the virtual environment that your application is running in.  Keep in mind that you are most likely building on Windows 10 Pro but your Container is running virtually on Linux.  Therefore, the command line interface will be for Linux and accepts just Linux commands.   To see a list of all the files in the Container you would issue the following three commands:

 

docker exec -it  71a /bin/bash

 

ls

 

exit

 

  

 

Step 9

 

Assume you will be making changes to your sample code as you progress in your college courses.  After you do, you will need to re-build after you test it locally then run it again as a Container.  The best way to accomplish this is to first stop your current Container.  You dont have to remove the image because the build process will reuse everything in the previous image that has not changed.  This, by design, makes the build run quickly.  If you only change code in a few files, only those files will be updated in the build.

 

docker stop 71a

 

 

Now you are ready to build your sample application again.

 

Step 10

 

For your courses final example, you will build your application again then will run two copies side-by-side as Docker Containers.  Since both applications use Port 8080 you will map each Container to a different Container Port without making any coding changes. 

 

Note:  Dont forget to substitute your dockerID for the one shown and dont forget the period at the end of the docker build command!

 

docker build -t pklein011/node-web-app .

 

Then start your first Container as:

 

docker run --name app1  -p 40150:8080 -d pklein011/node-web-app

 

Start your second Container as:

 

docker run --name app2  -p 40152:8080 -d pklein011/node-web-app

 

Then get a list of Containers and notice them both running but with different Ports mapped.

 

docker ps

 

 

Step 11

 

To test they are both running correctly you will contact each one on their specific Port number.

 

localhost:40150

 

 

localhost:40152

 

 

You should now have the basics to create a Container for your sample application as well as any application you will be using after you graduate.  There are some representative commands set up for you to use in the command Generate section of the Shortcuts.

 

Next, you will see how to apply these Containers to Clusters and manage them as one logical entity using Kubernetes.  This will all be done in the next college course in the series.
