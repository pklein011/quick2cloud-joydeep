Implementing Continuous Delivery for your Cloud developed application

 

Create the Development Toolchain

 

You are going to repeat exactly the same process you did for creating the Production Toolchain – but this time you are going to do that for the Development Toolchain.

 

Note:  The moment you configured your Toolchain for Production, immediately after the Create button was pushed it went into action.  Your newly created Production Toolchain began the process of Deploying your application in the IBM Cloud.  

 

Step 6

Login again to the IBM Cloud at www.ibm.com/cloud and display the IBM Dashboard.  Click on the top left  image and click DevOps from the list.

 

Step 7

 

Click on Create a Toolchain on the far top right of the display

 

Step 8

 

Click on the Develop a Cloud Foundry app Toolchain from the list.  This one has the basic structure needed for the course

 

Step 9

 

Change the Toolchain name in the setting sheet to reflect development.  Don’t forget the name must be unique so add your ibmID along with the word development at its end.   Also, don’t forget to type in your github.com repository name as shown below.

 

Click on Create to generate the Toolchain Key.  Then click on Create again to finish the Toolchain’s creation.

 

 

Tip:  Very Important! – Do not confuse your IBMid with your gitID.  They can be the same or different and if different you must make sure you use the gitID whenever you specify a Github.com repository.  Use IBMid in the Toolchain name.  If you make them the same ID it really becomes a non-issue.

 

A moment of pause...

 

Note: If your Development Toolchain fails to start, most likely the cause is that you ran over your resource quota for IBM’s Lite (free) plan.  Look at the Deploy logs (from the Deploy tile) and it will tell you why the Deploy failed.  If it’s a quota problem, stop all other running applications (excluding the Production Toolchain and IBM Services).  Restart the Deploy by clicking on the Play Button icon on the Deploy tile.

 

Let us take a moment to look back at what has been accomplished.  Currently, you have two Toolchains running your application in the IBM Cloud and each Toolchain contains an Issues tool, Toolchain repository, Eclipse editor, Git Editor and Pipeline Delivery tool that contains a Build and Deploy feature.  These two Toolchains can be seen on the IBM Dashboard under DevOps, the Toolchain names are:  Go ahead, take a look.

 

Note: when looking for your Toolchains, if you don’t see them make sure your Resource Group on the display says Default.

 

1.       simple-toolchain-IbmID-production

2.       simple-toolchain-ibmID-development 

 

fyi

 

If all was successful and no typos were made, those two Toolchains were automatically started when the Toolchains were created and will show up as running in the IBM Dashboard under Cloud Foundry.  Drill down to the application in each Toolchain’s details display (by clicking on each Toolchain’s name) and take the URL link at the top of that display to see your homepage.   If a Toolchain failed to start it may be because you are over your memory quota for the IBM Lite Plan.  Stop any non-toolchain applications you have running (excluding IBM Services) then restart the Toolchain that failed.  You should have enough memory to run at least two Toolchains.  You start the Toolchain by clicking on the three dots on the far right of the Toolchain Name row.

 

Take note of the Production and Development URLs in each of your sample application’s home page.  Write them down, for now, we will keep it that funny name until later.  Use these two URLs to view your applications when using a web browser. 

 

Note:  Toolchain URL names change each time you push new updates to your Toolchain repository.  That issue will be dealt with shortly but for now just take note of that.

 

Whew!

 

Be proud, you have just crossed the second milestone of setting up for Continuous Delivery and it will be well worth the effort.  Just remember, once you set this methodology up; you are done with it for a while.  When enhancing your sample application by adding features, you just keep reusing these two Toolchains; never having to fiddle with them until you want to explore more.
