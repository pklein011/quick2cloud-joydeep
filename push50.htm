Add Push Notifications to Android Mobile Devices

 

IBM Push Notification and Configuration

 

In the last section you got your Android Application configured to point to the IBM Push Notification Service.  You will now get the push notification service to point to Google’s Firebase project.  Basically, you are providing the two components with credentials so they can talk with each other and get your notification delivered.

 

Step 0

 

Find the Firebase credentials you saved for your Quick2Cloud Project but if you can’t remember where they are stored then you will have to get them from Firebase.  Perform Step 0a if you don’t remember where the credentials were saved.

 

Step 0a

 

Bring up your Firebase project from www.firebase.com.  Login (or you are already logged in) and go to console.  Click on your project q2cproject name. 

 

 

Click on the Project Overview setting’s icon then on Project settings

 

 

Then select the Cloud Messaging tab from the display to see your credentials.

 

 

You will be wanting to use the Server key and the Sender ID

 

Step 1

 

Log into the IBM Cloud.  Select your Push Notification Service then click on Push Notifications.

 

 

Once you click on Push Notifications the following should appear.

 

 

Cut and Paste your Firebase credentials into to fields above and save them.  You have now configured IBM Push Notification to talk with Google’s Firebase.  As you can see, you have almost completed the configuration shown below.  The last two pieces, those in black and green, are required for your Quick2Cloud Sample Web Application to send a push notification to your real Android device.  But before we continue with those last two steps lets try out the system and see if you can send a push notification message to your Android Emulator.  That will validate that your configuration is correct.

 

What has been accomplished so far…  Everything in gray has been done

 

 

Step 2

 

As part of the Gradle Sync process, Android Studio also did a Build of your IBM Supplied Android App.  You can see the Build results toward the bottom of the Android Studio display as shown below.  Make sure there are no severe errors.

 

 

If you make changes to the sample android app code, for example, to fix an error, you should Rebuild your application by the following drop-down display:

 

 

Now, you will Run your IBM Supplied Android Application in your selected emulator.  You begin this process by issuing the Run MainActivity command from the following display drop-down:

 

 

Once running, Android Studio will start your mobile device emulator and pass control to it.  At first, your emulator will start up just like a real mobile device, splash screen and all, then Android Studio will copy your IBM Supplied Android App to the emulator and start it running.  This entire process can take up to a minute so have some patients.  Part of the reason for the delay has to do with your virus checker looking over the emulator.  Android Studio will give you a pop-up message on how to make things run faster should you desire.  All messages from the emulator will show up in the Android Studio under the Run tab in the bottom window as shown below:

 

 

Notice too that the emulator looks just like the mobile device you previously created.   If everything was configured correctly between the IBM Supplied Android App and IBM’s Push Notification, you should see a message indicating your device has been successfully registered with IBM’s Push Notification (as shown in yellow above).

 

Step 3

 

It is time to fire off your first push notification.  Returning to the IBM Push Notification Service in the IBM Cloud Dashboard, click on the Basic CVQUS name and you will be displaying the opening display.  Click on Push Notifications

 

 

Select Messages from the menu and then type in your push message.  In this example the message is Quick2Cloud to the rescue…  When you have finished the message click on Send

 

 

At this point, the IBM Push Notification Service is attempting to deliver your message over to Google Firebase and then down to your emulator by passing through all the components that you have painstakingly configured.  And with a bit of good look and an eye for cleaning up any configuration typos – a message should be waiting for you on your mobile emulator.

 

 

Now, you should feel a sense of accomplishment running through your veins!  What you did was no easy task to learn but now that you have done it; it will be easier the second time around.  The hard part is over, we will now focus on work supplying your Quick2Cloud Sample Web Application with some code to make your application push notifications to your mobile device.
