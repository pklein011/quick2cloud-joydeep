Git Editor

 

Another editor available to you from the Toolchain in the IBM Cloud is the Git Editor.  Being very different than Eclipse; it works one-on-one with the Toolchain Repository editing its files directly out of that repository.  The Git Editor does not have a Workspace Repository or a Workspace Directory concept so you can’t test directly out of a working directory. 

 

With the git editor, every modified file saved is at the same time committed to the Toolchain Repository which makes developing a bit difficult.  With this methodology, it would be hard to build multi-file features since the moment you saved the first file of the feature the Toolchain’s Pipeline Delivery would deploy the application before the other files were updated.  Also, the only way to test your changes with the git editor is running it live and that won’t be a good choice when editing on the Production Toolchain.

 

Note:  To get around this Pipeline Delivery issue you can configure the Build stage in the Pipeline Tool to not trigger on a commit.  But, this defeats of the automation Continuous Delivery provides.

 

This is really not the behavior we want when developing application features in this course.  Eclipse is better suited for that purpose.  At least you can test your changes first using the Eclipse Editor before going live.   But, in the end it is your choice.  Use whichever you feel comfortable with.

 

Tip: Quick2Cloud recommends using the Eclipse Editor on all Toolchain editing, both Production and Development

 

Creating the Toolchains

 

Important: You should stop all your running applications in the IBM Cloud prior to creating Toolchains.  This is done to make sure, under IBM Cloud’s Lite (free) Plan, that you will not go over quota when the Toolchains start.  Go to the running applications on the IBM Dashboard and issue a Stop to them via the three dot drop-down on the far right of each application name.

 

Let us put aside all this reading for a moment and start the Continuous Delivery process with some hands-on experience by creating a Toolchain in the IBM Cloud.

 

Step 0

Login to the IBM Cloud at www.ibm.com/cloud and display the IBM Dashboard.  Click on the top left  image and click DevOps from the list.

 

 

 

Step 1

 

Click on Create a Toolchain on the far top right of the display

 

 

Step 2

 

Click on the Develop a Cloud Foundry app Toolchain from the list.  This one has the basic structure needed for the course

 

 

Step 3

 

Next, IBM’s Toolchain template will appear and you will be filling in a few fields before the Toolchain is complete.  Specifically, the fields that are highlighted in yellow in the diagram below:

 

For the Toolchain name, it needs to be unique and the one IBM generated for you (with that crazy large random string) is a unique name and can be used.  However, it does not help to understand what the Toolchain does.  Make sure your ibmID is somewhere in the name to help make it unique. 

 

Change the name to something like

 

simple-toolchain-ibmID-production 

 

For the source repository, remember to substitute in your Github.com repository name for gitID.

 

https://github.com/gitID/q2c-web-app.git

 

And yes, you do need the https:// in front of the Github.com URL or the Toolchain won’t find it.

 

Note: Toolchains are uniquely pre-named with a very long numerical string.   Change that name to something you understand

 

Just overtype on the name’s numerical part to make it look more like what is shown below.  Failing to do this will cause you to go crazy trying to figure out what each Toolchain belongs too.

 

Then click on Create to continue...

 

 

Step 4

 

After clicking on Create, Immediately you will be prompted for the Toolchain API Key that will be generated by the Toolchain for you.  Press Create to generate a key

 

 

Note: A Toolchain API Key is used instead of your password when the Toolchain is running.  You don’t want your password imbedded in any process that could be compromised by someone unauthorized looking over your shoulder.

 

Step 5

 

IBM’s Toolchain then generates you a key that binds the Toolchain to you.  Press Create at the right hand bottom (or top) again and this will finally finish the Toolchain creation and make it yours.

 

 

Congratulations!  You now have the Production Toolchain created and ready to give you the Continuous Delivery experience promised to you in this course.  Let us take a moment to reflect on what has been accomplished. 

 

You have read up on how the Toolchain works and the tools it provides.  For your Production work you have created a Toolchain that contains:

 

1.       Issues Tool

2.       Toolchain Git repository

3.       Eclipse Editor

4.       Git Editor

5.       Pipeline Delivery Tool

 

All of which are ready for you to use.

 

Great job!
