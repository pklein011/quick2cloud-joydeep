Clustering Containers using Kubernetes

 

Kubernetes

 

In preparing to use IBM Cloud’s Kubernetes you will need to upgrade to an IBM pay-as-you-go account.  Up until now you have been working with the IBM Cloud Free Plan which provides limited access to the Cloud but with the introduction of Kubernetes IBM requires a different Cloud Plan.  Fortunately, after signing up for your IBM pay-as-you-go account you still won’t be charged anything because in this course, you will be using IBM Cloud’s Free version of Kubernetes.  But, the only way to use the free version is to do the upgrade.

 

Step 0

 

Upgrade your account

 

 

Enter your credit card information and you will be ready to go.  Again, the IBM Cloud Kubernetes that you will be using in this course has no charge associated with it.  However, it is a limited version of Kubernetes, the Cluster you will be deploying only lasts 30 days, and it only contains 1 Worker Nodes so it is not extremely powerful.  But, for the purpose of understanding Kubernetes, it is perfect.  Should you ever go into production with a large application containerized in Kubernetes then you may consider upgrading to a more robust Cluster that comes with a small charge.

 

Tip:  For this course there are a lot of names you will be defining along the way; names you will need to recall at the end.  Therefore; it is highly recommended that you use a cheat sheet and jot down all names as you create them.  It will be less frustrating in the long run.

 

Step 1

 

You will now provision your Free IBM Cloud Kubernetes Cluster.  Login to the IBM Dashboard and select Kubernetes from the drop-down list on the far left.

 

 

Once you click on Kubernetes you will get the following display:

 

Click on Create Cluster.

 

 

Click on Free.

 

 

Once you click on the Free Cluster, then IBM Cloud will begin the task of provisioning the Cluster in the background.  Make sure you give the Cluster a name that you will remember.  Quick2Cloud suggests you append your ibmID in front of the name to uniquify it.  As an example, your Cluster name could be ibmID-mycluster.   Whatever name you decide, jot it down on a cheat sheet for you will need to recall it later. 

 

Provisioning a Cluster will take a bit of time, so, as the Cluster is provisioning, you will continue with the Kubernetes deployment by downloading the IBMCloud’s Kubernetes Commands.

 

Step 2

 

You will now download and install the IBMCloud Commands that deal with Kubernetes.  This is accomplished using the PowerShell CLI interface only.

 

Set-ExecutionPolicy Unrestricted; iex(New-Object Net.WebClient).DownloadString('http://ibm.biz/idt-win-installer')

 

 

Step 3

 

Before you can continue, you must wait for your Kubernetes Cluster to be fully provisioned.  How to tell when that happens requires you to login to the IBM Cloud Dashboard.  You will now see a new resource line containing Clusters; a line that previously was not there.   This line will contain the Cluster that is in the process of provisioning.

 

Click on the Cluster.

 

 

Click on the Cluster name being provisioned.

 

 

Click on the Worker Nodes tab.

 

 

You must wait in this display until the Status field says Normal. 

 

WAIT  Use the browser’s refresh feature to update the display.

 

Step 4

 

Now that the Cluster has been fully provisioned, login to the IBMCloud in preparation for issuing commands to configure it.  Use your ibmcloud userID in place of what is shown.

 

ibmcloud login -a cloud.ibm.com -r us-south -g Default -u ibmid@gmail.com

 

 

Enter in your IBMCloud password.

 

Step 5

 

IBM will provide you with the initialization information for the Cluster.  You will obtain that information by using the ks parameter on the IBMCloud command which communicates with IBMCloud’s Kubernetes and its Clusters.

 

ibmcloud ks cluster config --cluster bni1nnqd0lqo6uf6d85g

 

 

You must cut and paste the environment variable that is returned to you from the Cluster command.  Notice you can use either PowerShell or the DOS Prompt, just pick the one of the consoles you want to copy then paste it into.

 

 

Step 6

 

Verify that your Desktop can communicate with your fully provisioned Cluster.

 

kubectl version --short

 

 

Notice that you used the kubectl command to get the Kubernetes Cluster version.  It will be this command, combined with the IBMCloud ks command that will allow you, in the future, to manipulate the capabilities of your Kubernetes Cluster.
